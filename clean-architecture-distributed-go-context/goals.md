# Criterios de Validación (Goals)

- Cumple con Clean Architecture: separación estricta de capas
- DDD aplicado: agregados, entidades, servicios de dominio
- Outbox Pattern implementado y probado
- CQRS: comandos y queries separados
- Event-Driven: eventos integrados y auditados
- Repositorios híbridos: GORM/pgxpool
- Observabilidad: métricas, logs, health checks
- Docker Compose funcional para desarrollo local
- Tests unitarios y de integración presentes
- Documentación generada y actualizada
